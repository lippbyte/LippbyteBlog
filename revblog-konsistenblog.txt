1. MASALAH YANG DITEMUKAN
================================================================

Saat kamu membuka homepage (home/index.html) dan blog post
(blog/[topik]/index.html), ada kemungkinan tampilannya terasa
"berbeda nuansanya" — warna tidak pas, ukuran teks sedikit
berbeda, atau dark mode tidak sama persis.

Penyebabnya: ada DUA file CSS yang mendefinisikan variabel
warna yang HAMPIR SAMA tapi tidak identik:

  File 1: home/assets/css/style.css
  File 2: blog/template/css/style.css

Contoh perbedaan konkret yang ditemukan:

  Variabel         | home/style.css    | blog/style.css
  -----------------|-------------------|------------------
  --accent-soft    | #fff5f2           | #fff1ee
  --accent-mid     | #fdd8cc           | #ffd4c8
  --surface-2      | #f5f4f1           | #f2f0ec
  --border         | #e8e6e1           | #e6e2db
  --nav-h          | 58px              | 56px

Kelihatannya kecil, tapi efeknya nyata:
- Tombol di home terasa sedikit lebih terang dari tombol di blog
- Dark mode di home dan blog punya nuansa yang tidak sama
- Kalau kamu edit satu file, file lain tidak ikut berubah
  → kamu harus edit 2 tempat setiap kali ada perubahan desain

2. KONSEP DASAR YANG PERLU DIPAHAMI
================================================================

---- 2A. Apa itu CSS Custom Properties (CSS Variables)? --------

CSS Custom Properties adalah "variabel" di dalam CSS.
Kamu mendefinisikan nilai satu kali, lalu pakai di mana saja.

Cara mendefinisikan:
  :root {
    --warna-utama: #e8441a;
    --ukuran-font: 16px;
  }

  ":root" artinya: "berlaku untuk seluruh halaman".
  Nama variabel HARUS dimulai dengan dua tanda strip (--)

Cara memakainya:
  button {
    background: var(--warna-utama);
    font-size: var(--ukuran-font);
  }

  "var(--nama-variabel)" artinya: ambil nilai dari variabel itu.

Kenapa ini penting?
  Tanpa variabel:
    .btn      { background: #e8441a; }
    .badge    { background: #e8441a; }
    .link:hover { color: #e8441a; }
    → kalau warna mau diganti, harus cari & ganti di 3 tempat

  Dengan variabel:
    :root     { --accent: #e8441a; }
    .btn      { background: var(--accent); }
    .badge    { background: var(--accent); }
    .link:hover { color: var(--accent); }
    → ganti satu baris di :root, semua ikut berubah ✓


---- 2B. Apa itu Dark Mode dengan CSS? -------------------------

Dark mode diimplementasikan dengan menambahkan class "dark"
ke elemen <html>, lalu mendefinisikan ulang variabel-variabelnya.

  /* Default (light mode) */
  :root {
    --bg: #ffffff;
    --text: #000000;
  }

  /* Dark mode: timpa variabel yang sama */
  html.dark {
    --bg: #121212;
    --text: #f3f4f6;
  }

Cara kerjanya:
  1. CSS membaca :root dulu → set warna terang
  2. Kalau <html> punya class="dark" → variabel ditimpa
  3. Semua elemen yang pakai var(--bg) otomatis berubah

JavaScript yang toggle dark mode:
  document.documentElement.classList.toggle('dark');
  → documentElement = elemen <html>
  → classList.toggle('dark') = tambah class "dark" kalau belum ada,
     hapus kalau sudah ada


---- 2C. Apa itu "Single Source of Truth"? ---------------------

Ini adalah prinsip pemrograman:
"Setiap informasi hanya boleh ada di SATU tempat."

Analogi: Bayangkan kamu punya dua buku catatan yang berisi
nomor telepon teman-temanmu. Kalau salah satu teman ganti nomor,
kamu harus update DUA buku. Pasti ada yang lupa, dan dua buku
jadi tidak sinkron.

Di CSS kita: variabel warna ada di DUA file → rawan tidak sinkron.
Solusinya: SATU file variabel, di-link dari semua halaman.


---- 2D. Apa itu @import di CSS? -------------------------------

@import memungkinkan satu file CSS memasukkan file CSS lain.

  /* Di dalam style.css */
  @import url('./tokens.css');

  Ini artinya: "sebelum memproses file ini,
  muat dulu file tokens.css dari folder yang sama."

Alternatif lain: pakai dua tag <link> di HTML:
  <link rel="stylesheet" href="/shared/tokens.css" />
  <link rel="stylesheet" href="/home/style.css" />


================================================================
3. DIAGNOSIS: KENAPA TIDAK KONSISTEN?
================================================================

Mari kita trace alur lengkapnya:

SKENARIO A — Pembaca buka Homepage
  1. Browser muat home/index.html
  2. HTML meminta: <link href="assets/css/style.css">
  3. Browser muat home/assets/css/style.css
     → Di sini ada :root { --accent-soft: #fff5f2; ... }
  4. Semua elemen di homepage pakai variabel dari file ini

SKENARIO B — Pembaca buka Blog Post
  1. Browser muat blog/laravel-blog/index.html
     (file ini di-generate oleh CMS saat publish)
  2. HTML meminta: <link href="../template/css/style.css">
  3. Browser muat blog/template/css/style.css
     → Di sini ada :root { --accent-soft: #fff1ee; ... }
  4. Semua elemen di blog pakai variabel dari file ini

MASALAH:
  --accent-soft di homepage  = #fff5f2  (agak lebih terang)
  --accent-soft di blog post = #fff1ee  (agak lebih gelap)

  Ini akan terlihat di elemen seperti tombol hover, callout box,
  dan tag badge — yang semuanya pakai var(--accent-soft).

AKAR MASALAH:
  Kedua file dibuat "sendiri-sendiri" tanpa sinkronisasi.
  Ketika satu file diedit, file lain tidak ikut diupdate.


================================================================
4. SOLUSI: STRATEGI UNIFIKASI
================================================================

Ada 3 strategi yang bisa dipilih:

---- STRATEGI 1 (Direkomendasikan): File Tokens Terpisah --------

Buat satu file khusus untuk variabel:
  /shared/tokens.css  ← SATU-SATUNYA tempat variabel warna

Lalu di tiap CSS file, import file ini:
  home/assets/css/style.css   → @import '../../shared/tokens.css';
  blog/template/css/style.css → @import '../../../shared/tokens.css';

Kelebihan:
  ✓ Ubah satu file, semua halaman ikut berubah
  ✓ Mudah dibaca: variabel di satu tempat saja
  ✓ Skalabel: kalau ada halaman baru, tinggal import

Kekurangan:
  - Perlu path relatif yang benar (harus hati-hati ../../)
  - @import sedikit memperlambat loading (bisa diabaikan untuk blog)


---- STRATEGI 2: Copy-Replace Total ----------------------------

Pilih SATU versi variabel (pilih yang lebih lengkap), lalu
copy-paste ke file CSS yang lain.

Kelebihan:
  ✓ Paling sederhana, tidak ada dependency
  ✓ Tidak perlu pikir tentang path

Kekurangan:
  ✗ Tetap dua tempat → rawan tidak sinkron lagi di masa depan
  ✗ Tidak scalable


---- STRATEGI 3: Satu CSS File untuk Semua ---------------------

Gabungkan semua CSS menjadi satu file besar, taruh di root,
semua halaman pakai file yang sama.

Kelebihan:
  ✓ Benar-benar single source of truth
  ✓ Satu request ke server

Kekurangan:
  ✗ File jadi sangat besar
  ✗ Sulit di-maintain karena semua style tercampur

→ PILIHAN TERBAIK UNTUK LIPPBYTEBLOG: Strategi 1

---- LANGKAH 2: Edit home/assets/css/style.css -----------------

Hapus SEMUA baris yang ada di dalam :root { ... } dan html.dark { ... }.
Ganti dengan satu baris import di bagian PALING ATAS file:

```css
/* HAPUS semua ini:
:root {
  --bg: ...
  --surface: ...
  (semua variabel)
}
html.dark {
  (semua variabel dark mode)
}
*/

/* GANTI DENGAN: */
@import url('../../shared/tokens.css');

/* Sisanya tetap sama, yaitu semua kelas CSS: */
*, *::before, *::after { ... }
body { ... }

/* dst... */
```

Penjelasan path '../../shared/tokens.css':
  Kita sekarang berada di: home/assets/css/style.css
  Satu level ke atas (../)  = home/assets/
  Dua level ke atas (../../) = home/
  Tapi tokens.css ada di /shared/ (setingkat dengan /home/)
  Jadi pathnya: ../../shared/tokens.css
  Yang artinya: naik 2 level dari css/ lalu masuk ke shared/

  Struktur folder yang dimaksud:
  /
  ├── shared/
  │   └── tokens.css       ← file baru
  ├── home/
  │   └── assets/
  │       └── css/
  │           └── style.css ← dari sini, ../../shared/tokens.css
  └── blog/
      └── template/
          └── css/
              └── style.css ← dari sini, ../../../shared/tokens.css

Path '../../../shared/tokens.css':
  Kita di: blog/template/css/style.css
  ../       = blog/template/

---- LANGKAH 4: Sinkronisasi --nav-h di template HTML ----------

Di blog/template/index.html ada kode yang menggunakan --nav-h:
  pt-14  (Tailwind = padding-top 56px)

Tailwind pakai angka hardcoded, bukan CSS variable.
Kamu perlu ganti ke inline style agar mengikuti token:

```html
<!-- SEBELUM (hardcoded Tailwind) -->
<div class="... pt-14 ...">

<!-- SESUDAH (pakai CSS variable) -->
<div class="..." style="padding-top: var(--nav-h)">
```

Atau definisikan di Tailwind config:
```javascript
tailwind.config = {
  theme: {
    extend: {
      spacing: {
        'nav': 'var(--nav-h)'  /* sekarang bisa pakai pt-nav */
      }
    }
  }
}
```

---- LANGKAH 5: Update CMS Admin (admin/index.html) -----------

Di admin/index.html, ada bagian ini yang perlu diperhatikan:
  <link rel="stylesheet" href="../blog/template/css/style.css" id="previewCss" />

Ini untuk preview di panel CMS. Setelah tokens.css dibuat,
pastikan blog/template/css/style.css sudah mengimport tokens.css.
Kalau sudah, preview CMS akan otomatis menggunakan token yang sama.

Tapi ada satu masalah: CMS admin punya CSS sendiri di <style> tag:
  :root {
    --cms-bg: #0d1117;
    --cms-accent: #e8441a;
    ...
  }

Ini pakai prefix "--cms-" jadi tidak bentrok dengan "--bg" dll.
Tidak perlu diubah — ini memang variabel khusus untuk UI admin,
bukan untuk blog post.


================================================================
6. PENJELASAN TIAP BARIS KODE PENTING
================================================================

Berikut penjelasan mendalam untuk kode-kode kunci di project ini:

---- A. CSS Custom Properties (:root) --------------------------

  :root {
    --bg: #fafaf8;
  }

  - ":root" = selector untuk elemen paling atas di HTML (<html>)
  - Variabel di :root bisa diakses dari MANA SAJA di halaman
  - "#fafaf8" = kode warna hex (putih sedikit kekuningan/krem)
  - Nama "--bg" dipilih singkat karena dipakai berkali-kali


---- B. Kenapa Dark Mode Pakai html.dark bukan :root ? ----------

  html.dark {
    --bg: #111110;
  }

  Kenapa "html.dark" bukan ":root.dark" atau "body.dark"?

  Karena JavaScript toggle class di elemen <html>:
    document.documentElement.classList.toggle('dark')
    → documentElement = elemen <html>

  Jadi kalau class "dark" ada di <html>, selector "html.dark" aktif
  dan variabel di dalamnya MENIMPA variabel di :root.

  Urutan specificity CSS:
    :root { --bg: white; }      ← dibaca pertama
    html.dark { --bg: black; }  ← dibaca kedua, MENIMPA yang pertama
                                    (berlaku hanya kalau ada class dark)


---- C. @import di CSS -----------------------------------------

  @import url('../../shared/tokens.css');

  - "@import" = aturan CSS untuk memasukkan file CSS lain
  - "url(...)" = path ke file yang mau di-import
  - Path HARUS relatif dari lokasi file CSS yang mengimport
  - HARUS ditulis di baris PALING ATAS (sebelum rules CSS lain)

  Kenapa tidak direkomendasikan pakai @import di production?
  Karena browser harus download file pertama dulu, baru tahu
  bahwa ada file lain yang harus didownload (serial, bukan paralel).
  Tapi untuk project blog seperti ini, efeknya tidak terasa.

  Alternatif yang lebih cepat: dua tag <link> di HTML
    <link rel="stylesheet" href="/shared/tokens.css">
    <link rel="stylesheet" href="/home/style.css">
  Ini didownload paralel (bersamaan) → lebih cepat.


---- D. var() di CSS -------------------------------------------

  .btn {
    background: var(--accent);
    color: var(--text-primary, #000000);
  }

  - "var(--accent)" = ambil nilai dari variabel --accent
  - "var(--accent, fallback)" = kalau --accent tidak ada,
    gunakan "#000000" sebagai fallback
  - Fallback berguna untuk mencegah elemen "kosong" kalau
    ada typo di nama variabel


---- E. Sistem Warna Bertingkat --------------------------------

  Di tokens.css ada 4 level teks:
    --text-primary: #000000;  ← paling gelap, untuk judul penting
    --text-body:    #1a1a1a;  ← sedikit lebih terang, untuk isi teks
    --text-muted:   #4d4d4d;  ← lebih terang lagi, untuk teks sekunder
    --text-faint:   #737373;  ← paling terang, untuk teks hint/label

  Kenapa bertingkat?
  Supaya ada hierarki visual. Mata pembaca secara otomatis
  akan melihat teks gelap dulu, baru teks terang.

  Penggunaannya di CSS:
    .article-title { color: var(--text-primary); }  ← paling penting
    .article-body  { color: var(--text-body); }
    .article-date  { color: var(--text-muted); }    ← sekunder
    .form-hint     { color: var(--text-faint); }    ← paling tidak penting

  Di dark mode, semua ini dibalik:
    --text-primary: #f2f1ec;  ← terang di background gelap
    --text-faint:   #5a5750;  ← agak gelap (tidak terlalu kontras)


---- F. Variabel Shadow (Bayangan) -----------------------------

  --shadow-xs:   0 1px 2px rgba(0,0,0,.04);
  --shadow-card: 0 2px 12px rgba(28,26,22,.05), 0 1px 2px rgba(28,26,22,.04);

  Format shadow CSS: offset-x | offset-y | blur | spread | warna

  Kenapa nilainya sangat kecil (.04, .05)?
  Karena bayangan yang terlalu gelap terlihat "plastik" dan kuno.
  Desain modern pakai bayangan yang hampir tidak terlihat (subtle)
  untuk memberikan kesan kedalaman yang natural.

  Di dark mode, nilai opacity dinaikkan:
  --shadow-xs: 0 1px 2px rgba(0,0,0,.4);
  Karena di background gelap, bayangan kecil tidak terlihat.

8. CATATAN BELAJAR
================================================================

KONSEP BESAR YANG DIPELAJARI DI SINI:

1. Single Source of Truth
   → Setiap data/konfigurasi sebaiknya hanya ada di SATU tempat.
     Ini prinsip yang berlaku di CSS (variabel), JavaScript (state),
     database (normalisasi), bahkan di kehidupan nyata.

2. CSS Architecture (Arsitektur CSS)
   → Cara mengorganisir CSS agar mudah di-maintain.
     Ada beberapa metodologi terkenal: BEM, ITCSS, CSS Modules.
     Yang kita lakukan ini mirip dengan pendekatan "Design Tokens"
     yang dipakai perusahaan besar seperti Airbnb, GitHub, Spotify.

3. Design Token
   → Istilah industri untuk variabel-variabel desain (warna, ukuran,
     font, shadow) yang disimpan terpusat. Token ini kemudian bisa
     dipakai di website, aplikasi mobile, bahkan dokumen desain Figma.
     LippbyteBlog pakai konsep ini secara tidak langsung melalui
     CSS Custom Properties.

4. CSS Specificity (Kekhususan)
   → Kenapa html.dark bisa menimpa :root?
     Karena html.dark LEBIH SPESIFIK dari :root.
     CSS punya sistem penilaian: semakin spesifik selector,
     semakin prioritas aturannya.

5. Relative Path (Path Relatif)
   → Saat menulis @import url('../../shared/tokens.css'),
     kita pakai "relative path" — path yang dihitung dari lokasi
     file saat ini. Ini berbeda dari "absolute path" yang dihitung
     dari root server (misal: /shared/tokens.css).
     Relative path lebih portabel karena tidak bergantung pada
     lokasi deploy di server.
