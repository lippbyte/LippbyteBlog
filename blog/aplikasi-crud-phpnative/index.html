Native PHP: Implementasi CRUD dengan Relasi dan Validasi Input

2024-07-28 🏷 Praktik
Praktik ini membahas implementasi operasi CRUD (Create, Read, Update, Delete) pada PHP native dengan relasi dua tabel, yakni siswa dan kelas. Anda akan belajar mengelola data siswa yang terhubung dengan pilihan kelas dinamis dari tabel master. Selain itu, praktik ini juga mencakup penerapan validasi input pada form untuk memastikan integritas data dan memberikan peringatan kepada pengguna.
🎯 Tujuan Pembelajaran
  • Memahami konsep relasi antar tabel dalam aplikasi CRUD PHP native.
  • Mampu membuat operasi CRUD (Create, Read, Update, Delete) untuk data yang memiliki relasi.
  • Mampu mengimplementasikan validasi input pada form PHP native.
  • Mampu menampilkan pesan error atau sukses berdasarkan hasil validasi.

CRUD dengan Relasi Dua Tabel

Pada bagian ini, kita akan fokus membangun fungsionalitas CRUD dengan konsep relasi antar dua tabel: siswa dan kelas.

Persiapan Database

  • **Memastikan Tabel Kelas:** Pastikan Anda sudah memiliki tabel `kelas` yang berisi data master kelas.
  • **Mengosongkan Tabel Siswa:** Kosongkan tabel `siswa` menggunakan perintah `TRUNCATE`. **Perhatian!** Pastikan Anda menggunakan `TRUNCATE` dan bukan `DROP`.
  • **Mengubah Struktur Tabel Siswa:** Ubah tabel `siswa` dengan mengganti kolom `kelas` menjadi `id_kelas` dengan tipe data `INT`. **Perhatian!** Pastikan tabel `siswa` dalam keadaan kosong untuk menghindari potensi error saat mengubah struktur.

Memodifikasi Tampilan Daftar Siswa

  • **File `index.php`:** Lakukan perubahan kode pada file `index.php` untuk menampilkan data siswa beserta nama kelas dari tabel relasi. php // Contoh perubahan yang mungkin diperlukan di index.php // Query untuk mengambil data siswa dan nama kelas $query = "SELECT s.*, k.nama_kelas FROM siswa s JOIN kelas k ON s.id_kelas = k.id"; // ...
  • **File `views/v_index.php`:** Sesuaikan kode pada file `views/v_index.php` agar dapat menampilkan `nama_kelas` yang diambil dari tabel `kelas`. Perhatikan bahwa `nama_kelas` adalah nama field pada tabel `kelas`. html

Menambahkan Data Siswa

  • **File `tambah.php` (Mengambil Data Kelas):** Tambahkan kode untuk mengambil data kelas dari database sebelum menyertakan form view `v_tambah.php`. Data ini akan digunakan untuk mengisi opsi pilihan kelas pada form. php
  • **File `views/v_tambah.php` (Form Pilihan Kelas):** Ubah kode pada file `views/v_tambah.php` untuk menambahkan elemen `select` yang akan menampilkan daftar kelas secara dinamis. html
  • **File `tambah.php` (Proses Insert):** Sesuaikan kode pada file `tambah.php` untuk mengambil data `id_kelas` dari `POST` dan menambahkannya ke dalam query `INSERT` data siswa. php

Mengubah Data Siswa

  • **File `edit.php` (Proses Update):** Lakukan hal yang sama seperti pada proses `tambah.php`. Ambil data `id_kelas` dari `POST` dan masukkan ke dalam query `UPDATE` untuk data siswa. php
  • **File `edit.php` (Mengambil Data Kelas untuk Form):** Tambahkan query untuk mengambil data kelas (sama seperti di `tambah.php`) agar daftar pilihan kelas dapat ditampilkan pada form edit. Selain itu, pastikan untuk menandai kelas yang sedang aktif (yang dimiliki oleh siswa yang diedit). php >... include 'views/v_edit.php'; ?>

Validasi Input

Pada bagian ini, kita akan mengimplementasikan validasi input untuk setiap form guna memastikan data yang dimasukkan valid dan memberikan umpan balik kepada pengguna.

Implementasi Validasi pada Form Tambah Siswa

  • **File `views/v_tambah.php` (Menampilkan Pesan):** Tambahkan baris kode ini di atas form, atau sesuaikan posisinya dengan desain layout Anda, untuk menampilkan pesan sukses atau error. html
    0) : ?>
  • **File `tambah.php` (Logika Validasi):** Tambahkan dan ubah kode pada `tambah.php` untuk menyertakan logika validasi. Contoh ini menunjukkan validasi untuk `nis` dan `nama_lengkap`. Anda bisa memperluasnya untuk field lain. php

Penjelasan Mekanisme Validasi

  • **Menampilkan Pesan Sukses:** Baris kode untuk `if (isset($success) && $success != '')` digunakan untuk menampilkan pesan sukses yang berasal dari variabel `$success`. Jika variabel tersebut tidak bernilai kosong, maka tampilkan isi dari variabel tersebut dalam bentuk `alert-success`.
  • **Menangani Pesan Error:** Baris kode untuk `if (isset($errors) && count($errors) > 0)` digunakan untuk melakukan validasi. Misalnya, jika variabel `$nis` kosong, maka tambahkan pesan error ke array `$errors` dan tampilkan menggunakan function `flash` (jika Anda memiliki implementasi `flash message`) atau secara langsung di bagian atas form dalam bentuk `alert-danger` dengan daftar `
    • ` dari semua error.
    • **Lokasi Insert/Update Data:** Pastikan Anda menyimpan kode untuk melakukan operasi `INSERT` data ataupun `UPDATE` data pada blok `if (count($errors) == 0)` setelah semua validasi selesai dilakukan dan tidak ada error yang ditemukan.

    Alur Validasi Input

    Berikut adalah gambaran alur proses validasi input data:

    • **Start:** Proses validasi dimulai.
    • **NIS Kosong?** * Jika `Ya`: Tampilkan pesan error bahwa NIS kosong, lalu lanjutkan ke validasi berikutnya (atau hentikan dan tampilkan semua error). * Jika `Tidak`: Lanjutkan ke validasi berikutnya.
    • **Nama Lengkap Kosong?** * Jika `Ya`: Tampilkan pesan error bahwa Nama Lengkap kosong, lalu lanjutkan ke validasi berikutnya. * Jika `Tidak`: Lanjutkan ke validasi berikutnya.
    • **Field ke-N Kosong?** (Lanjutkan untuk semua field yang perlu divalidasi). * Jika `Ya`: Tampilkan pesan error bahwa field ke-N kosong. * Jika `Tidak`: Lanjutkan.
    • **Semua Validasi Sukses:** Jika tidak ada error yang terkumpul, lakukan operasi `Insert` atau `Update` Data ke Database.
    • **Finish:** Proses validasi dan penyimpanan data selesai.

Login & Authorization

2026-05-14 🏷 Praktik
Membuat sistem login agar hanya user yang terautentikasi yang bisa mengakses halaman data siswa. Password dienkripsi dengan SHA1 dan state login disimpan di PHP Session.
🎯 Tujuan Pembelajaran
  • Membuat tabel t_login dengan enkripsi SHA1
  • Membuat form login dengan validasi session
  • Membuat fungsi checkLogin() dan sudahLogin()
  • Mengamankan halaman dari akses tanpa login
  • Membuat fitur logout

Struktur Database

Buat tabel baru bernama t_login dengan struktur berikut:

sql
CREATE TABLE t_login (
  id_login INT(11) AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(100),
  password VARCHAR(100),
  level TINYINT(1)
);

Insert data awal (password dienkripsi SHA1 via phpMyAdmin Function):

sql
INSERT INTO t_login (username, password, level)
VALUES ('admin', SHA1('admin'), 1);
📝

Note ✨

**Catatan** Saat insert di phpMyAdmin, pilih Function = SHA1 pada kolom password, lalu ketik "admin" sebagai nilainya. Jangan ketik hash SHA1-nya manual!

Logika Login

login.php
<?php
include 'lib/library.php';
 
sudahLogin(); // Cegah user yang sudah login membuka halaman ini lagi
 
if ($_SERVER['REQUEST_METHOD'] == "POST") {
    $username = $_POST['username'];
    $password = $_POST['password'];
 
    $sql  = "SELECT * FROM t_login
             WHERE username = '$username'
             AND password = SHA1('$password')";
 
    $data = $mysql->query($sql) or die($mysql->error);
 
    if ($data->num_rows != 0) {
        $row = mysqli_fetch_object($data);
        $_SESSION['username'] = $row->username;
        $_SESSION['level']    = $row->level;
        header('location: index.php');
    } else {
        $error = "Username atau password salah";
    }
}
 
include 'views/v_login.php';
?>
☁️

Attention! 💥

**Perhatian** Password dibandingkan menggunakan SHA1() langsung di query SQL, sehingga cocok dengan data yang tersimpan di database.

Helper Function

File ini berisi fungsi-fungsi utilitas yang dipakai di seluruh aplikasi.

lib/helper.php
<?php
session_start();
 
// Ganti URL sesuai folder project kamu
function base_url() {
    return "http://localhost/nama_project_kamu";
}
 
// Fungsi Flash Message (simpan/ambil pesan ke session)
function flash($tipe, $pesan = '') {
    if (empty($pesan)) {
        $pesan = @$_SESSION[$tipe];
        unset($_SESSION[$tipe]);
        return $pesan;
    } else {
        $_SESSION[$tipe] = $pesan;
    }
}
 
// Cek apakah user sudah login — jika belum, redirect ke login
function checkLogin() {
    $username = @$_SESSION['username'];
    $level    = @$_SESSION['level'];
    if (empty($username) AND empty($level)) {
        header("location: login.php");
    }
}
 
// Kebalikan checkLogin — jika sudah login, jangan buka login lagi
function sudahLogin() {
    $username = @$_SESSION['username'];
    $level    = @$_SESSION['level'];
    if (!empty($username) AND !empty($level)) {
        header("location: index.php");
    }
}
?>
📝

Note ✨

**Catatan** File helper.php harus di-include di library.php dengan baris: include 'lib/helper.php'; Pastikan session_start() hanya dipanggil SATU KALI di seluruh aplikasi (biasanya di helper.php). Jangan panggil session_start() di file lain.

Searching, Ordering & Sorting

Searching, Ordering & Sorting

2026-05-15 🏷 Praktik
Menambahkan fitur pencarian data berdasarkan NIS atau nama lengkap, serta fitur sorting kolom secara Ascending/Descending dengan klik ikon panah pada header tabel.
🎯 Tujuan Pembelajaran
  • Membuat form pencarian dengan metode GET
  • Menggunakan SQL LIKE untuk pencarian fleksibel
  • Menambahkan parameter sort dan order ke URL
  • Menggunakan ORDER BY pada query SQL secara dinamis

Form Pencarian

Tambahkan form search di atas tabel data:

views/v_index.php
<form method="GET" action="index.php">
  Cari Berdasarkan NIS dan NAMA
  <input type="text" name="search" value="<?= @$search ?>">
  <button type="submit">Cari</button>
</form>
📝

Note ✨

**Catatan** Method GET digunakan agar kata kunci pencarian tampil di URL, sehingga bisa di-bookmark atau dibagikan. Atribut value="" berguna untuk menampilkan kembali kata yang sudah diketik user setelah form disubmit.\

Logika Pencarian

Penjelasan `LIKE %$search%`

Tanda %di kiri dan kanan berarti "mengandung kata tersebut di posisi manapun". Contoh: LIKE %khalifa% akan menemukan Khalifa, Muhammad Khalifa, alifkhalifa15, dll.

index.php
<?php
include 'lib/library.php';
checkLogin();
 
$sql = 'SELECT * FROM siswa';
 
// --- SEARCHING ---
$search = @$_GET['search'];
if (!empty($search)) {
    $sql .= " WHERE nis LIKE '%$search%'
              OR nama_lengkap LIKE '%$search%' ";
}
 
// --- ORDERING ---
$order_field = @$_GET['sort'];   // Kolom yang akan diurutkan
$order_mode  = @$_GET['order'];  // ASC atau DESC
 
if (!empty($order_field) && !empty($order_mode)) {
    $sql .= " ORDER BY $order_field $order_mode";
}
 
$listSiswa = $mysqli->query($sql);
 
include 'views/v_index.php';
?>
☁️

Attention! 💥

**Perhatian** Kode di atas rentan SQL Injection untuk proyek produksi. Untuk belajar, ini sudah cukup. Untuk produksi, gunakan Prepared Statement (mysqli_prepare / PDO).

Tambahkan link panah ▲▼ di header tabel untuk tiap kolom yang ingin bisa disortir:

Upload File

2026-05-16 🏷 Teori
Menambahkan fitur upload foto ke form tambah dan edit data siswa. Foto disimpan di folder media/images/ dan nama filenya dicatat di kolom "file" pada tabel siswa.
🎯 Tujuan Pembelajaran
  • Menambahkan kolom file ke tabel siswa
  • Menggunakan enctype="multipart/form-data" pada form
  • Memproses upload dengan $_FILES dan move_uploaded_file()
  • Menangani kondisi edit tanpa mengganti foto
  • Mengubah batas ukuran upload di php.ini

Alter Table Siswa

Tambahkan kolom "file" untuk menyimpan nama file foto:

sql
ALTER TABLE t_siswa ADD COLUMN file VARCHAR(100);

Struktur akhir tabel t_siswa:

Buat Folder untuk menyimpan file

Buat folder baru di dalam project kamu:

php
nama_project/

Attention! 💥

**Perhatian** Folder ini harus ada sebelum upload dilakukan, karena PHP tidak otomatis membuat folder. Jika folder tidak ada, upload akan gagal.

**Catatan** enctype="multipart/form-data" WAJIB ada jika form memiliki input bertipe file. Tanpa ini, file tidak akan terkirim ke server.

**Catatan** Input type="hidden" name="foto" menyimpan nama file lama agar ketika user edit data tapi tidak ganti foto, foto lama tidak hilang.

Proses Upload

tambah.php
<?php
include 'db/koneksi.php';
include 'lib/lib.php';
 
if ($_SERVER['REQUEST_METHOD'] == 'POST') {
    $nis            = @$_POST['nis'];
    $nama_lengkap   = @$_POST['nama_lengkap'];
    $jenis_kelamin  = @$_POST['jenis_kelamin'];
    $alamat         = @$_POST['alamat'];
    $no_telp        = @$_POST['no_telp'];
    $kelas          = @$_POST['id_kelas'];
    $foto           = $_FILES['foto'];     // Ambil data file dari $_FILES
 
    // Cek apakah user mengupload file?
    if (!empty($foto) AND $foto['error'] == 0) {
        $path   = "./media/images/";
        $upload = move_uploaded_file($foto['tmp_name'],
                                     $path . $foto['name']);
 
        if (!$upload) {
            flash('error', "Upload file gagal");
            header('location: index.php');
            exit;
        }
        $file = $foto['name'];
    }
 
    $sql = "INSERT INTO t_siswa
            (nis, nama_lengkap, jenis_kelamin, alamat, no_telp, id_kelas, file)
            VALUES
            ('$nis','$nama_lengkap','$jenis_kelamin','$alamat','$no_telp','$kelas','$file')";
 
    if ($mysql->query($sql)) {
        flash('success', "Data berhasil ditambahkan.");
    } else {
        $error = "Error. " . $mysql->error;
    }
}
 
// Ambil data kelas untuk dropdown
$sql        = "SELECT * FROM kelas";
$dataKelas  = $mysqli->query($sql) or die($mysqli->error);
 
$form = "add";
$url  = "tambah.php";
include 'views/v_form_siswa.php';
?>

Bootstrap Modal & Ajax

Bootstrap Modal & Ajax

2026-05-17 🏷 Praktik
Mengganti konfirmasi hapus biasa (browser confirm) dengan Bootstrap Modal yang lebih elegan. Proses hapus dilakukan via Ajax tanpa reload halaman, dan notifikasi sukses ditampilkan menggunakan library Toastr.
🎯 Tujuan Pembelajaran
  • Memahami cara kerja Bootstrap Modal
  • Menampilkan Modal konfirmasi sebelum hapus data
  • Melakukan request AJAX dengan jQuery $.ajax()
  • Menggunakan library Toastr untuk notifikasi
  • Memodifikasi delete.php untuk mendukung AJAX

Tambahkan Dependency

views/v_index.php
// jQuery & Bootstrap JS 
<script type="text/javascript"
  src="<?= base_url() ?>/media/js/jquery.min.js"></script>
<script type="text/javascript"
  src="<?= base_url() ?>/media/js/bootstrap.min.js"></script>
 
// Toastr (notifikasi) 
<link href="<?= base_url() ?>/media/plugins/toastr/toastr.min.css"
      rel="stylesheet">
<script type="text/javascript"

Javascript untuk Modal & Ajax

views/v_index.php
<script type="text/javascript">
$(function() {
 
  // Tangkap event klik pada tombol .btnDelete
  $(".btnDelete").on("click", function(e) {
    e.preventDefault(); // Batalkan aksi href default
 
    // Ambil nama dari kolom ke-2 di baris yang diklik
    var nama = $(this).parent().parent().children()[2];
    nama = $(nama).html();
 
    // Simpan baris (tr) yang diklik
    var tr = $(this).parent().parent();
 
    // Ambil URL dari atribut href tombol
    var href = $(this).attr('href');
 
    // Tampilkan modal
    $(".modal").modal('show');
    $(".modal-title").html('Konfirmasi');
    $(".modal-body").html('Anda yakin ingin menghapus data <b>' + nama + '</b> ?');
 
    // Hapus event lama, pasang event baru pada tombol "Ya"
    $('.btnYa').off();
    $('.btnYa').on('click', function() {
      $.ajax({
        'url'    : href,
        'type'   : 'POST',
        'success': function(result) {
          if (result == 1) {
            $(".modal").modal('hide');
            tr.fadeOut(); // Sembunyikan baris yang dihapus
            toastr.success('Data berhasil dihapus', 'Informasi');
          }
        }
      });
    });
 
  });
 
});
</script>

Penjelasan Alur:

  • User klik tombol Delete → muncul Modal konfirmasi
  • User klik "Ya" → Ajax POST ke delete.php
  • delete.php memproses dan return angka 1 (sukses) atau 0 (gagal)
  • Jika result == 1 → baris tabel di-fadeOut + notifikasi Toastr

Modifikasi delete.php — return 1 atau 0

delete.php
<?php
include 'db/koneksi.php';
 
$nis = $_GET['nis'];
if (!empty($nis)) {
    $sql = "DELETE FROM t_siswa WHERE nis = '$nis'";
 
    if ($mysql->query($sql)) {
        echo 1; // Sukses → digunakan oleh Ajax
    } else {
        echo 0; // Gagal
    }
}
?>
☁️

Attention! 💥

**Perhatian** Setelah diubah untuk Ajax, delete.php tidak lagi melakukan header redirect ke index.php. Return value-nya hanya angka 1 atau 0 yang dibaca oleh JavaScript.

Catatan angka kolom di JavaScript:

  • $(this).parent().parent().children()[2]
  • Angka [2] merujuk ke kolom ke-3 (index dari 0). Sesuaikan
  • angkanya dengan posisi kolom nama di tabel kamu!

CRUD Relasi 2 Tabel

CRUD Relasi 2 Tabel

2026-05-18 🏷 Praktik
Membangun relasi antara tabel siswa dan tabel kelas, sehingga pilihan kelas pada form tambah/edit bersifat dinamis — diambil dari database, bukan ditulis hardcode.
🎯 Tujuan Pembelajaran
  • Memahami konsep relasi tabel (Foreign Key)
  • Menggunakan INNER JOIN untuk menggabungkan dua tabel
  • Membuat dropdown dinamis dari database
  • Menggunakan ternary operator untuk selected option
  • Menyertakan id_kelas dalam INSERT dan UPDATE

Buat Tabel Kelas

sql
CREATE TABLE kelas (
  id_kelas   INT(11) AUTO_INCREMENT PRIMARY KEY,
  nama_kelas VARCHAR(15),
  jurusan    VARCHAR(50)
);
 
-- Insert beberapa data contoh
INSERT INTO kelas (nama_kelas, jurusan) VALUES
  ('XI RPL 1', 'Rekayasa Perangkat Lunak'),
  ('XI RPL 2', 'Rekayasa Perangkat Lunak'),
  ('XII RPL 1', 'Rekayasa Perangkat Lunak');

Modifikasi Tabel Siswa

Langkah-langkah:

  • TRUNCATE tabel siswa dulu (kosongkan data): Di phpMyAdmin → tab Operations → "Empty the table (TRUNCATE)"
☁️

Attention! 💥

**perhatian** TRUNCATE hanya mengosongkan data, tabel tetap ada. DROP menghapus tabel beserta strukturnya secara permanen!
  • ALTER TABLE — ganti kolom kelas menjadi id_kelas:
sql
ALTER TABLE t_siswa
  CHANGE kelas id_kelas INT(11);

Query dengan INNER JOIN

Ubah query di index.php untuk menyertakan data dari tabel kelas:

index.php
<?php
include 'db/koneksi.php';
 
$success = flash('success');
$error   = flash('error');
 
cekLogin();
 
// Gunakan INNER JOIN untuk menggabungkan data siswa dan kelas
$sql  = 'SELECT * FROM siswa
         INNER JOIN kelas ON (siswa.id_kelas = kelas.id_kelas)';
$data = $mysql->query($sql) or die($mysql->error);
 
include 'views/v_index.php';
?>

Tampilkan Nama Kelas di Tabel

Karena sudah di-JOIN, gunakan nama_kelas (dari tabel kelas):

views/v_index.php
<td><?= $siswa['nis'] ?></td>
<td><?= $siswa['nama_lengkap'] ?></td>
<td><?= $siswa['jenis_kelamin'] ?></td>
<td><?= $siswa['nama_kelas'] ?></td>   <!-- bukan $siswa['kelas'] lagi! -->
<td><?= $siswa['jurusan'] ?></td>
☁️

Attention! 💥

**Perhatian** nama_kelas adalah nama field dari tabel kelas (bukan tabel siswa). Jika masih menggunakan $siswa['kelas'], akan muncul error undefined.

Dropdown Kelas Dinamis

Di tambah.php, ambil data kelas sebelum include view:

tambah.php
// Ambil data kelas untuk ditampilkan di dropdown
$sql        = "SELECT * FROM kelas";
$dataKelas  = $mysqli->query($sql) or die($mysqli->error);
 
$form = "add";
$url  = "tambah.php";
include 'views/v_form_siswa.php';

Di view form (views/v_form_siswa.php), buat dropdown:

views/v_form_siswa.php
<div class="form-group">
  <label>Kelas</label>
  <select name="id_kelas" class="select1">
    <option value="">[ Pilih Kelas ]</option>
    <?php while ($murid = @$dataKelas->fetch_array()) { ?>
      <option value="<?= $murid['id_kelas'] ?>"
        <?= @$siswa['id_kelas'] == $murid['id_kelas'] ? 'selected' : '' ?>>
        <?= $murid['nama_kelas'] ?>
      </option>
    <?php } ?>
  </select>
</div>

Penjelasan logika selected:

  • Baris ternary operator mengecek apakah id_kelas siswa sama dengan
  • id_kelas pada iterasi loop. Jika sama → tambahkan atribut selected
  • agar opsi tersebut tampil terpilih saat mode edit.

Simpan id_kelas saat insert

tambah.php
$nis            = $_POST['nis'];

// ...proses upload foto...
 
$sql = "INSERT INTO t_siswa
        (nis, nama_lengkap, jenis_kelamin, alamat, no_telp, id_kelas, file)
        VALUES
        ('$nis','$nama_lengkap','$jenis_kelamin','$alamat','$no_telp','$kelas','$file')";

Update id_kelas saat edit

Di edit.php, tambahkan pengambilan data kelas untuk dropdown, dan sertakan id_kelas di query UPDATE:

$sql        = "SELECT * FROM t_kelas";

edit.php
// Dalam blok POST:
$kelas = $_POST['id_kelas'];
 
$sql = "UPDATE t_siswa SET
        nama_lengkap  = '$nama_lengkap',
        jenis_kelamin = '$jenis_kelamin',
        alamat        = '$alamat',
        no_telp       = '$no_telp',
        id_kelas      = '$kelas',    -- ← sertakan di UPDATE
        file          = '$file'
        WHERE nis = '$nis'";
