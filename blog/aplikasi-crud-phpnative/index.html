Native PHP: Implementasi CRUD dengan Relasi dan Validasi Input

2024-07-28 🏷 Praktik
Praktik ini membahas implementasi operasi CRUD (Create, Read, Update, Delete) pada PHP native dengan relasi dua tabel, yakni siswa dan kelas. Anda akan belajar mengelola data siswa yang terhubung dengan pilihan kelas dinamis dari tabel master. Selain itu, praktik ini juga mencakup penerapan validasi input pada form untuk memastikan integritas data dan memberikan peringatan kepada pengguna.
🎯 Tujuan Pembelajaran
  • Memahami konsep relasi antar tabel dalam aplikasi CRUD PHP native.
  • Mampu membuat operasi CRUD (Create, Read, Update, Delete) untuk data yang memiliki relasi.
  • Mampu mengimplementasikan validasi input pada form PHP native.
  • Mampu menampilkan pesan error atau sukses berdasarkan hasil validasi.

CRUD dengan Relasi Dua Tabel

Pada bagian ini, kita akan fokus membangun fungsionalitas CRUD dengan konsep relasi antar dua tabel: siswa dan kelas.

Persiapan Database

  • **Memastikan Tabel Kelas:** Pastikan Anda sudah memiliki tabel `kelas` yang berisi data master kelas.
  • **Mengosongkan Tabel Siswa:** Kosongkan tabel `siswa` menggunakan perintah `TRUNCATE`. **Perhatian!** Pastikan Anda menggunakan `TRUNCATE` dan bukan `DROP`.
  • **Mengubah Struktur Tabel Siswa:** Ubah tabel `siswa` dengan mengganti kolom `kelas` menjadi `id_kelas` dengan tipe data `INT`. **Perhatian!** Pastikan tabel `siswa` dalam keadaan kosong untuk menghindari potensi error saat mengubah struktur.

Memodifikasi Tampilan Daftar Siswa

  • **File `index.php`:** Lakukan perubahan kode pada file `index.php` untuk menampilkan data siswa beserta nama kelas dari tabel relasi. php // Contoh perubahan yang mungkin diperlukan di index.php // Query untuk mengambil data siswa dan nama kelas $query = "SELECT s.*, k.nama_kelas FROM siswa s JOIN kelas k ON s.id_kelas = k.id"; // ...
  • **File `views/v_index.php`:** Sesuaikan kode pada file `views/v_index.php` agar dapat menampilkan `nama_kelas` yang diambil dari tabel `kelas`. Perhatikan bahwa `nama_kelas` adalah nama field pada tabel `kelas`. html

Menambahkan Data Siswa

  • **File `tambah.php` (Mengambil Data Kelas):** Tambahkan kode untuk mengambil data kelas dari database sebelum menyertakan form view `v_tambah.php`. Data ini akan digunakan untuk mengisi opsi pilihan kelas pada form. php
  • **File `views/v_tambah.php` (Form Pilihan Kelas):** Ubah kode pada file `views/v_tambah.php` untuk menambahkan elemen `select` yang akan menampilkan daftar kelas secara dinamis. html
  • **File `tambah.php` (Proses Insert):** Sesuaikan kode pada file `tambah.php` untuk mengambil data `id_kelas` dari `POST` dan menambahkannya ke dalam query `INSERT` data siswa. php

Mengubah Data Siswa

  • **File `edit.php` (Proses Update):** Lakukan hal yang sama seperti pada proses `tambah.php`. Ambil data `id_kelas` dari `POST` dan masukkan ke dalam query `UPDATE` untuk data siswa. php
  • **File `edit.php` (Mengambil Data Kelas untuk Form):** Tambahkan query untuk mengambil data kelas (sama seperti di `tambah.php`) agar daftar pilihan kelas dapat ditampilkan pada form edit. Selain itu, pastikan untuk menandai kelas yang sedang aktif (yang dimiliki oleh siswa yang diedit). php >... include 'views/v_edit.php'; ?>

Penjelasan Mekanisme Validasi

  • **Menampilkan Pesan Sukses:** Baris kode untuk `if (isset($success) && $success != '')` digunakan untuk menampilkan pesan sukses yang berasal dari variabel `$success`. Jika variabel tersebut tidak bernilai kosong, maka tampilkan isi dari variabel tersebut dalam bentuk `alert-success`.
  • **Menangani Pesan Error:** Baris kode untuk `if (isset($errors) && count($errors) > 0)` digunakan untuk melakukan validasi. Misalnya, jika variabel `$nis` kosong, maka tambahkan pesan error ke array `$errors` dan tampilkan menggunakan function `flash` (jika Anda memiliki implementasi `flash message`) atau secara langsung di bagian atas form dalam bentuk `alert-danger` dengan daftar `
    • ` dari semua error.
    • **Lokasi Insert/Update Data:** Pastikan Anda menyimpan kode untuk melakukan operasi `INSERT` data ataupun `UPDATE` data pada blok `if (count($errors) == 0)` setelah semua validasi selesai dilakukan dan tidak ada error yang ditemukan.

    Alur Validasi Input

    Berikut adalah gambaran alur proses validasi input data:

    • **Start:** Proses validasi dimulai.
    • **NIS Kosong?** * Jika `Ya`: Tampilkan pesan error bahwa NIS kosong, lalu lanjutkan ke validasi berikutnya (atau hentikan dan tampilkan semua error). * Jika `Tidak`: Lanjutkan ke validasi berikutnya.
    • **Nama Lengkap Kosong?** * Jika `Ya`: Tampilkan pesan error bahwa Nama Lengkap kosong, lalu lanjutkan ke validasi berikutnya. * Jika `Tidak`: Lanjutkan ke validasi berikutnya.
    • **Field ke-N Kosong?** (Lanjutkan untuk semua field yang perlu divalidasi). * Jika `Ya`: Tampilkan pesan error bahwa field ke-N kosong. * Jika `Tidak`: Lanjutkan.
    • **Semua Validasi Sukses:** Jika tidak ada error yang terkumpul, lakukan operasi `Insert` atau `Update` Data ke Database.
    • **Finish:** Proses validasi dan penyimpanan data selesai.
