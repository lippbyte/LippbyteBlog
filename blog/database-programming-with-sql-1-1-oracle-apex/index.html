Database Programming with SQL 1-1 Oracle APEX

📅 2020-01-01⏱ ~7 menit baca🏷 Teori
Dokumen ini memperkenalkan konsep perangkat lunak aplikasi dan sistem, serta membahas penggunaan Oracle APEX sebagai lingkungan praktik untuk mempelajari sintaks SQL. Pembaca akan diajak untuk memahami cara masuk ke Oracle APEX, menjalankan kueri SQL dasar untuk mengambil data, dan menerapkan aturan SQL untuk menampilkan data tertentu dari database.
🎯 Tujuan Pembelajaran
  • Membedakan antara perangkat lunak aplikasi dan perangkat lunak sistem beserta contohnya.
  • Masuk ke lingkungan praktik Oracle APEX.
  • Menjalankan kueri sederhana untuk mengambil informasi dari database.
  • Menerapkan aturan SQL untuk menampilkan semua kolom dan subset kolom yang ditetapkan berdasarkan kriteria.

Setiap hari, kita berinteraksi dengan berbagai aplikasi komputer, mulai dari memeriksa email hingga berbelanja. Aplikasi ini dirancang untuk mempermudah tugas pengguna tanpa perlu memahami detail teknis di baliknya. Dalam kursus ini, kita akan mempelajari sintaks SQL menggunakan aplikasi yang disebut Oracle APEX.

Perangkat Lunak Aplikasi vs. Perangkat Lunak Sistem

Perangkat lunak aplikasi adalah program yang memungkinkan pengguna akhirâorang-orang seperti kitaâuntuk melakukan tugas-tugas spesifik. Ini adalah program yang sepenuhnya dikembangkan dan siap digunakan, menyembunyikan kompleksitas internal dari pengguna. Contoh perangkat lunak aplikasi meliputi program pemrosesan kata, database, game, email, dan grafis. Sebagai analogi, aplikasi bekerja seperti mobil; pengguna hanya perlu tahu cara mengemudikannya, tanpa perlu memahami cara kerja transmisi atau mesin secara mendalam.

Sebaliknya, perangkat lunak sistem terdiri dari program tingkat rendah yang dirancang untuk berinteraksi dengan perangkat keras komputer. Perangkat lunak ini adalah fondasi yang memungkinkan perangkat lunak aplikasi berjalan. Contoh perangkat lunak sistem termasuk sistem operasi (OS), kompilator, dan utilitas sistem.

Banyak situs web besar, seperti Yahoo.com, menggunakan database Oracle untuk menyimpan data. Aplikasi yang mereka sediakan telah memprogram semua SQL dan bahasa pengodean lainnya di muka, sehingga pengguna dapat mengakses informasi hanya dengan beberapa klik.

Oracle APEX: Lingkungan Pengembangan dan Pembelajaran SQL

Oracle APEX (Application Express) adalah aplikasi yang akan digunakan dalam kursus ini. Ini memungkinkan banyak pengembang untuk membuat dan mengakses aplikasi seolah-olah aplikasi tersebut berjalan di database terpisah. Oracle APEX mempercepat proses pengembangan aplikasi berkat fitur bawaan seperti tema desain, kontrol navigasi, pengendali bentuk, dan laporan yang fleksibel.

Oracle APEX memiliki dua komponen utama:

  • **SQL Workshop**: Digunakan untuk mempelajari dan menjalankan perintah SQL.
  • **Application Builder**: Digunakan untuk merancang dan membangun aplikasi.

Dalam konteks kursus ini, fokus utama adalah pada SQL Workshop untuk memungkinkan pengguna membuat tabel dan mengambil informasi dari database Oracle. Familiaritas dengan SQL sangat penting untuk dapat menemukan subset data berdasarkan kriteria pencarian tertentu dengan cepat.

💡

Catatan

**Catatan Penting**: Akun Oracle APEX yang baru diberikan biasanya kosong tanpa tabel atau data. File skrip yang disediakan dalam sumber kursus perlu dijalankan untuk menambahkan tabel dan data yang akan digunakan selama pembelajaran. Untuk informasi lebih lanjut, Panduan Pelajar dan Instruktur Oracle APEX dapat diakses melalui sumber kursus.

Dasar-dasar Pernyataan SQL dengan Oracle APEX

Untuk berinteraksi dengan database menggunakan SQL, ada beberapa perintah dasar yang perlu dipahami:

Melihat Semua Kolom dari Tabel

Perintah SELECT * digunakan untuk mengembalikan semua baris dan semua kolom yang ada di sebuah tabel.

Sintaks: SELECT * FROM <nama_tabel>;

Contoh: SELECT * FROM employees;

Melihat Subset Kolom dengan Kriteria Tertentu

Untuk mengembalikan hanya subset data atau kolom tertentu dari sebuah tabel berdasarkan kriteria, gunakan perintah SELECT dengan menyebutkan nama kolom dan menambahkan klausa WHERE.

Sintaks: SELECT <nama_kolom_1, nama_kolom_2, ...> FROM <nama_tabel> WHERE <kondisi>;

Contoh: SELECT first_name, last_name, job_id FROM employees WHERE job_id = 'SA_REP';

Penanganan Kesalahan dalam SQL

Ketika memasukkan perintah SQL, sangat penting untuk menggunakan ejaan yang benar dan sintaks yang tepat, baik untuk perintah itu sendiri, nama tabel, maupun nama kolom. Kesalahan umum dan cara memperbaikinya meliputi:

  • **Kesalahan Sintaks Perintah**: Jika perintah SQL salah dieja (misalnya, `SELCT` alih-alih `SELECT`), sistem akan menghasilkan pesan kesalahan. Solusinya adalah memperbaiki ejaan perintah dan menjalankannya kembali.
  • **Kesalahan Nama Tabel**: Jika nama tabel yang dimasukkan salah (misalnya, `employee` alih-alih `employees`), pesan kesalahan akan muncul. Perbaiki ejaan nama tabel agar sesuai dengan yang ada di database.
  • **Kesalahan Nama Kolom**: Demikian pula, jika nama kolom salah dimasukkan (misalnya, `name` alih-alih `first_name`), sistem akan menunjukkan kesalahan. Pastikan nama kolom yang digunakan benar dan sesuai dengan definisi tabel.

Terminologi Penting

Berikut adalah istilah-istilah penting yang digunakan dalam pelajaran ini:

  • **Perangkat lunak aplikasi**: Program yang dirancang untuk pengguna akhir melakukan tugas spesifik.
  • **Perangkat lunak sistem**: Program tingkat rendah yang berinteraksi dengan hardware komputer.
  • **Oracle APEX**: Aplikasi yang digunakan untuk pengembangan dan pembelajaran SQL.
  • **Sintaks**: Aturan penulisan perintah dalam bahasa pemrograman atau kueri.
  • **Subset**: Bagian dari keseluruhan data atau informasi.
  • **Operator Perbandingan**: Simbol yang digunakan dalam klausa `WHERE` untuk membandingkan nilai (contoh: `=`, `>`, `<`, dll.).

Refleksi

Renungkan materi ini:

  • Memahami perbedaan mendasar antara perangkat lunak aplikasi dan sistem adalah kunci untuk menghargai peran aplikasi seperti Oracle APEX dalam mempermudah interaksi pengguna dengan sistem kompleks seperti database. Familiaritas dengan SQL melalui APEX memberdayakan pengguna untuk mengakses dan memanipulasi data secara efisien.
